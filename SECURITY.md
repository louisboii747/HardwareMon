# Security Policy

## Supported Versions

HardwareMon is an actively developed project. Security updates are only provided for the latest stable release.

| Version               | Supported      |
| --------------------- | -------------- |
| Latest stable release | ✅              |
| Older releases        | ❌              |
| Development builds    | ⚠️ Best effort |

If you discover a security issue, please ensure you are using the latest version before reporting it.

---

## Reporting a Vulnerability

The security of HardwareMon and its users is taken seriously. If you believe you have discovered a security vulnerability, please report it privately rather than creating a public GitHub issue.

### How to report

Please send an email to:

**[louishinchliffe11@gmail.com](mailto:louishinchliffe11@gmail.com)**

Include as much information as possible, such as:

* A description of the vulnerability
* Steps to reproduce the issue
* The affected operating system and HardwareMon version
* Screenshots, logs, or proof-of-concept code (if applicable)
* Any suggested mitigation or fix

### What to expect

After receiving your report:

* You should receive an acknowledgement within **72 hours**.
* I will investigate the report and may contact you for additional information.
* If the vulnerability is confirmed, I will work on a fix as quickly as possible.
* Once a fix is available, a new release will be published and the vulnerability will be disclosed responsibly where appropriate.

### Responsible Disclosure

Please do **not** publicly disclose security vulnerabilities until a fix has been released. Responsible disclosure helps protect HardwareMon users while the issue is being resolved.

Thank you for helping make HardwareMon safer for everyone.
